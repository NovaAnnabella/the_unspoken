Introduction

Le thème du testing semble encore aujourd’hui être un terrain vierge avec une très grande liberté d’interprétation. La traditionnelle pyramide de test a été remise en question et de nouvelles pyramides de test ont vu le jour. A mon avis, il ne faut pas de pyramide de test, mais une compréhension claire de ce qui doit être testé. Les tests de niveau inférieur sont souvent moins significatifs. L’accent devrait être mis avant tout sur les tests de comportement, afin de s’assurer que la API ou l’UI fonctionne comme prévu.
